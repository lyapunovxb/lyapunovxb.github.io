2-Qwen模型镜像制作

发表于2025-11-06|更新于2025-11-06|模型训练调优

由于新机器挂载文件存储速度非常慢，通过dockerfile文件来直接生成镜像非常慢，所以本镜像在自己的V100云主机中进行打包。

  1. 首先docker pull拉取一个ubuntu基础环境

https://hub.docker.com/r/nvidia/cuda/tags?page=11&page_size=&name=&ordering=

<font style="color:#000000;">docker pull nvidia/cuda:12.2.2-cudnn8-devel-ubuntu22.04</font>

  1. 安装nvidia-container-toolkit，使docker可以调用宿主机gpu资源
    1. <font style="color:#000000;">distribution=$(. /etc/os-release;echo $ID$VERSION_ID) && \ curl -fsSL </font>[<font style="color:#000000;">https://nvidia.github.io/libnvidia-container/gpgkey</font>](https://nvidia.github.io/libnvidia-container/gpgkey)<font style="color:#000000;"> | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg && \ curl -s -L </font>[<font style="color:#000000;">https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list</font>](https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list)<font style="color:#000000;"> | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list</font>
    2. 安装nvidia-container-toolkit

<font style="color:#000000;">sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit</font>

安装过程中如果发现<font style="color:#000000;">Unable to locate package nvidia-container-toolkit</font>

添加nvidia-docker源

<font style="color:#000000;">curl -s -L </font>[<font style="color:#000000;">https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list</font>](https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list)<font style="color:#000000;"> | sudo tee /etc/apt/sources.list.d/nvidia-docker.list</font>

更新并重新执行安装

sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit

如果执行过程中报W: GPG error: https://nvidia.github.io/libnvidia-container/stable/ubuntu18.04/amd64 InRelease: The following signatures couldn’t be verified because the public key is not available: NO_PUBKEY DDCAE044F796ECB0

则需要确认你的系统是否信任NVIDIA的GPG密钥。如果没有，你需要导入它。可以通过以下命令导入GPG密钥：

curl -s [https://nvidia.github.io/libnvidia-container/gpgkey](https://nvidia.github.io/libnvidia-container/gpgkey) | sudo apt-key add -

3. <font style="color:#000000;">完成 nvidia-container-toolkit 的安装之后，我们继续执行 nvidia-ctk runtime configure 命令，为 Docker 添加 nvidia 这个运行时。完成后，我们的应用就能在容器中使用显卡资源了</font>

<font style="color:#000000;">sudo nvidia-ctk runtime configure --runtime=docker</font>

4. 重启docker

sudo systemctl restart docker

5. 查看是否安装成功

dpkg -l | grep nvidia-container-toolkit

  1. 拉取基础镜像成功后，创建一个docker容器

docker images

sudo docker run -it --name qwen --gpus all nvidia/cuda:12.2.2-cudnn8-devel-ubuntu22.04

exit

  1. 退出容器后，将本地跑同的qwen模型代码/权重/数据集/环境cp到创建的qwen镜像中

docker cp /opt/tmp/Qwen/ 02649afd9710:/qwen

  1. 重启docker，exec执行

<font style="color:rgba(0, 0, 0, 0.85);">docker ps -a</font>

<font style="color:rgba(0, 0, 0, 0.85);">docker start qwen</font>

<font style="color:rgba(0, 0, 0, 0.85);">docker exec -it 02649afd9710 bash</font>

  1. 因为想在容器中执行自身的python环境，不借用宿主机的环境，所以需要单独再安装conda、pytorch等环境。
  2. 安装完基础环境后，需要安装qwen模型的依赖

pip install -r requirements.txt

  1. 没有vim编辑器还需安装vim

<font style="color:rgba(0, 0, 0, 0.85);">apt-get update</font>

<font style="color:rgba(0, 0, 0, 0.85);">apt-get install -y vim</font>

  1. deepspeed安装

pip install "peft<0.8.0" deepspeed

  1. 此时在qwen容器中执行训练脚本，拉起训练。

bash finetune/finetune_lora_single_gpu.sh

  1. 将此时qwen容器打成镜像

docker commit -a "wangxiangbo" -m "qwen 7B" 02649afd9710 qwen-7b:v1.0

  1. 将打好的镜像转成tar包，供新机器解压使用

docker save -o qwen-7b.tar qwen-7b:v1.0

  1. 加载tar镜像, 使用load进行从tar文件导出镜像

<font style="color:#000000;">docker load -i qwen-7b.tar</font>

  1. 由于新机器挂载文件存储的原因，镜像解压速度太慢，于是打算将打好的qwen镜像push到阿里云个人仓库中，在新机器中直接pull该镜像。

  1. 将镜像推送到Registry

docker login --username=aliyun9599911612 registry.cn-shanghai.aliyuncs.com wxb121227

docker tag 37c7b97b67f6 registry.cn-shanghai.aliyuncs.com/aliyun_repository_wxb/repository_wxb:v1.0

docker push registry.cn-shanghai.aliyuncs.com/aliyun_repository_wxb/repository_wxb:v1.0

  1. 在3号机器中，拉取该镜像

docker pull registry.cn-shanghai.aliyuncs.com/aliyun_repository_wxb/repository_wxb:v1.0

  1. 通过该镜像，run一个容器

docker run -it --name qwen --gpus all registry.cn-shanghai.aliyuncs.com/aliyun_repository_wxb/repository_wxb:v1.0 bash
