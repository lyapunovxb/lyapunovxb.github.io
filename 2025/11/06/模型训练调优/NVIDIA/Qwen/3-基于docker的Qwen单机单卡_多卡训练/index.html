3-基于docker的Qwen单机单卡_多卡训练

发表于2025-11-06|更新于2025-11-06|模型训练调优
  1. 查看容器

docker ps

  1. exec进入容器

docker exec -it containerid bash

  1. 进入qwen目录并修改finetune_lora_single_gpu.sh参数
  1. 执行finetune_lora_single_gpu.sh单机单卡
  2. 修改finetune_lora_ds.sh参数
  1. 修改deepspeed中ds_config_zero2.json配置文件，增加TFlops显示

其中具体增加的参数配置为flops_profiler

  1. 执行finetune_lora_ds.sh单机多卡训练

其中报了Error while creating shared memory segment /dev/shm/nccl-KXWrmA (size 9637888)导致在docker中单机多卡拉起失败

问题原因：docker的shm共享内存不足，可以通过命令

<font style="color:rgb(77, 77, 77);">df -h | grep shm</font>查看当前容器的shm大小，默认为64M，这是远远不够的，所以要增加该容器的shm共享内存大小。参考博文https://blog.csdn.net/gg864461719/article/details/112466585

解决方法1：创建完容器之后，手动修改shm共享内存大小

a. 首先要关闭docker, 否则下面的操作步骤会无效.

<font style="color:rgb(77, 77, 77);">service docker stop</font>

b. 进入宿主机中/docker/containers/容器id 修改该容器的hostconfig.json文件，把其中的ShmSize的大小后面增加22（就变为了6.3G）其默认的是67108864KB 就约等于64M。

c. 重启docker服务

<font style="color:rgb(0, 0, 0);">systemctl start docker</font>

d. 解决完之后，重新查看shm的共享内存大小，此时已经变为了6.3G

解决方法2：在通过镜像run容器时，就直接指定–****shm-size 6G

<font style="color:rgb(77, 77, 77);">docker run -it --name qwen --gpus all --shm-size 6G registry.cn-shanghai.aliyuncs.com/aliyun_repository_wxb/repository_wxb:v1.0 bash</font>

之后重新拉起训练bash finetune/finetune_lora_ds.sh

其中tflops在9.5~10.5之间
