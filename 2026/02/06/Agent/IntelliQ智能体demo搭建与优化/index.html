IntelliQ智能体demo搭建与优化

发表于2026-02-06|更新于2026-02-06|AgentIntelliQ

项目说明

官方介绍

IntelliQ 是一个开源项目，旨在提供一个基于大型语言模型（LLM）的多轮问答系统。该系统结合了先进的意图识别和词槽填充（Slot Filling）技术，致力于提升对话系统的理解深度和响应精确度。

github

https://github.com/answerlink/IntelliQ

项目功能描述

  • 多轮对话管理：能够处理复杂的对话场景，支持连续多轮交互。
  • 意图识别：准确判定用户输入的意图，支持自定义意图扩展。
  • 词槽填充：动态识别并填充关键信息（如时间、地点、对象等）。
  • 接口槽技术：直接与外部APIs对接，实现数据的实时获取和处理。

缺陷解决

相同场景下插槽的更新问题

在用户第二次询问qwen2-7B的训练情况时，正常情况是，如果判断为同一个场景，应该先清除当前场景插槽的所有参数，然后通过轮询和根据用户的输入重新填入插槽项，而不是直接将上一个插槽的内容直接填充，因为qwen-7B的集群在哈尔滨1集群，并且我想看的是损失值而非tflops。

prompt增加相关场景所需参数的示例输入

prompt增加相关场景所需参数的示例输入引导输入，增加鲁棒性。避免用户的非法输入，导致智能体不稳定。

多轮回答函数修正

模型训练和推理性能场景增加了相同的job_id作为唯一身份认证，当在相同场景下，如果用户输入了不同的job_id，则智能体需要将本轮的场景插槽清空，并且将场景插槽中的job_id更改为当前的job_id，同时进一步询问用户缺失参数。

增加提取job_id函数

增加获取get和设置set插槽指定key的值

由于插槽slot返回的是一个列表[]，非字典{}，无法通过self.slot.get(“job_id”)直接获取到当前的job_id值，故需要增加相关get和set以遍历列表的方式获取指定job_id的值，并动态设置其值。

在插槽处理中增加清空插槽函数，重置为空置状态

修改prompt，每轮只向用户提问一个参数

由于场景判断和相关性计算都是依靠大模型来进行的，存在一定的不确定的性，回答的效果很大程度上依赖于LLM对于prompt给出的回答，当prompt设置为每轮向用户提问两个及以上参数时，LLM容易将用户的输入内容分析错误。
